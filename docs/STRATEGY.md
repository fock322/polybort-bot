## Как работает бот — подробное объяснение

### Что такое Polymarket?

Polymarket — это платформа предсказательных рынков (prediction markets), где люди торгуют бинарными контрактами на исходы реальных событий. Каждый рынок задаёт вопрос вроде «Победит ли Трамп на выборах 2028?» или «Достигнет ли Bitcoin $150,000 к концу 2026?». У каждого рынка есть два токена: **YES** (да, событие произойдёт) и **NO** (нет, не произойдёт). Цена токена — от $0.01 до $0.99 и отражает вероятность исхода (цена $0.65 = 65% вероятность). Если событие происходит, YES-токен погашается за $1.00, а NO — за $0.00 (и наоборот).

### Что делает бот?

Бот ищет **несоответствия** между «истинной» вероятностью события и рыночной ценой. Если бот считает, что реальная вероятность 75%, а рынок торгует на 65%, есть **edge** (преимущество) в 10¢. Бот покупает YES-токены по $0.65, ожидая, что рынок приблизится к истинной цене.

### Эволюция стратегии

| Версия | Название | Подход | Проблема |
|--------|----------|--------|----------|
| v1.0 (Python) | Fixed Spread | Маркет-мейкинг с фиксированным спредом | Не адаптируется к рынку |
| v2.0 (Python) | Dynamic Spread | Динамический спред + тренд (EMA) | Слабая адаптация |
| v3.0 (TypeScript) | Adaptive Alpha | Односторонние котировки + тренд + стоп-лосс | Adverse selection (плохой выбор) |
| v4.0 (TypeScript) | Adaptive Alpha ULTRA | Улучшенная v3.0 с большим числом рынков | Маркет-мейкинг убыточен на информированных рынках |
| **v5.0 (TypeScript)** | **Informed Edge Trading** | Оценка вероятности → Edge → Kelly → One-sided | **Текущая** |

**Ключевой переход v4→v5**: Маркет-мейкинг (двусторонние котировки) умирает от adverse selection — информированные трейдеры торгуют против тебя. v5.0 полностью отказывается от маркет-мейкинга и переходит к **информированному трейдингу**: оцениваем вероятность, находим edge, торгуем одну сторону.

## Стратегия v5.0 Informed Edge Trading

### Конвейер принятия решений (каждые 60 секунд)

```
1. ПОЛУЧИТЬ данные рынков (Gamma API, до 300 рынков)
   ↓
2. ОТФИЛЬТРОВАТЬ: ликвидность > $5,000, объём > $10,000, спред < 15¢, вероятность 5-95%
   → Остаётся ~30 лучших рынков
   ↓
3. ДЛЯ КАЖДОГО РЫНКА: получить стакан (CLOB API)
   ↓
4. ОЦЕНИТЬ истинную вероятность (5 факторов):
   ├── Depth-weighted price (35%) — VWAP стакана, глубже сторона тянет цену
   ├── Time decay (20%) — близость к сеттлменту, цена → край
   ├── Volume profile (20%) — высокий объём = доверяем рынку
   ├── Cross-market signals (25%) — корреляция между рынками (пока заглушка)
   └── LLM override (40% вес) — AI-оценка вероятности
   ↓
5. РАССЧИТАТЬ edge = |наша_оценка - рыночная_цена| × 100
   ↓
6. ФИЛЬТР: edge > 5¢? → НЕТ → SKIP
   ↓
7. РАЗМЕР ПОЗИЦИИ: Quarter Kelly (¼ от оптимального f)
   ├── Kelly fraction = (b×p - q) / b
   ├── Применяем 25% от Kelly (консервативно)
   ├── Макс. $30 на рынок
   ├── Мин. $5 на сделку
   └── 20% банкролла в резерве
   ↓
8. ИСПОЛНЕНИЕ: односторонний лимитный ордер (BUY_YES или BUY_NO)
   ├── VWAP-исполнение по стакану (частичное заполнение)
   ├── Phantom liquidity: 10% уровней стакана исчезают
   ├── Латентность: 2-8 секунд (цена двигается)
   └── API сбои: 5% rate limit, 2% server error, 1% timeout
   ↓
9. ВЫХОД из позиции (5 условий):
   ├── Edge reversal: оценка развернулась на 3¢+ → ВЫХОД
   ├── Edge narrow: edge < 2¢ → ВЫХОД
   ├── Time exit: > 48 циклов (~48 часов) → ВЫХОД
   ├── Settlement avoid: < 24ч до сеттлмента → ВЫХОД (избегаем 2% комиссию)
   └── HOLD: edge валидный → держим
```

### Оценка вероятности (детально)

#### 1. Depth-Weighted Price (вес 35%)
Ключевая инновация. Вместо простого mid-price (среднее между лучшим bid и ask), вычисляется **объёмно-взвешенная цена** с учётом глубины стакана. Если bid-сторона глубже (больше ордеров), цена тянется к bid; если ask-сторона — к ask. Это даёт более точную оценку «истинной» цены, чем mid-price.

```
depthWeightedPrice = (bidVWAP × bidDepth + askVWAP × askDepth) / (bidDepth + askDepth)
```

#### 2. Time Decay (вес 20%)
Близость к дате сеттлмента влияет на цену:
- **< 1 день**: цена тянется к 0 или 1 (событие почти resolved) — сдвиг ±5%
- **< 7 дней**: умеренный сдвиг к краям — ±2%
- **> 180 дней**: высокая неопределённость — тянем к 50/50 (±15%)

#### 3. Volume Profile (вес 20%)
Высокий объём торгов = больше информации = цена точнее:
- **> $100K**: доверяем цене полностью
- **> $50K**: 95% цены + 5% к 50%
- **> $20K**: 90% цены + 10% к 50%
- **< $20K**: 85% цены + 15% к 50% (низкая информативность)

#### 4. Cross-Market Signals (вес 25%)
Корреляция между связанными рынками (пока заглушка — return base price).
Планируется: если «Трамп победит» растёт, то и «Республиканцы победят» должен расти.

#### 5. LLM Override (вес 40% когда доступен)
Вызов AI-модели через `z-ai-web-dev-sdk` для оценки вероятности на основе:
- Вопроса рынка
- Текущей цены, глубины стакана, спреда
- Объёма, ликвидности, даты окончания

LLM вызывается только если количественный edge > 3¢ (не тратим ресурсы на неинтересные рынки). Результат кэшируется на 5 циклов.

**Итоговая оценка**:
- Без LLM: `final = quantitative` (4 фактора × веса)
- С LLM: `final = quantitative × 60% + llm × 40%`

### Kelly Criterion (размер позиции)

Kelly Criterion определяет оптимальный размер ставки для максимизации лог-роста банкролла:

```
f* = (b × p - q) / b

где:
  f* = доля банкролла
  b  = шансы (odds) = (1 - price) / price
  p  = наша оценка вероятности
  q  = 1 - p
```

Пример: YES-токен стоит $0.65, наша оценка 75% (edge = 10¢)
- b = (1 - 0.65) / 0.65 = 0.538
- p = 0.75, q = 0.25
- f* = (0.538 × 0.75 - 0.25) / 0.538 = 0.285 = 28.5% банкролла

**Мы используем ¼ Kelly** (7.1% банкролла) для консервативности — это значительно снижает волатильность при минимальной потере доходности.

### Модель фрикции (реалистичное моделирование)

Бот моделирует реальные препятствия торговли:

| Фактор | Значение | Описание |
|--------|----------|----------|
| Rate limit | 5% | API возвращает 429 |
| Server error | 2% | API возвращает 502/503 |
| Timeout | 1% | Запрос висит 30с |
| Empty book | 3% | Стакан пустой |
| Phantom liquidity | 10% | Уровни стакана исчезают |
| Латентность | 2-8с | Случайная задержка |
| Газ | $0.015/ордер | Стоимость транзакции |
| Слиппаж | VWAP | Исполнение по худшей цене |
| Комиссия сеттлмента | 2% | На выигрыш при погашении |
| Высокая волатильность | ×3 failure rate | Увеличивает все сбои |

### Circuit Breaker

При просадке > 25% от начального депозита торговля автоматически останавливается.

## Мини-сервисы

### 1. Data Collector (Port 3004)

**Файл**: `mini-services/data-collector/index.ts`

Собирает реальные данные с Polymarket для бэктестинга и анализа.

| Параметр | Значение |
|----------|----------|
| Порт | 3004 |
| Интервал | 20 секунд |
| Максимум рынков | 200 |
| Мин. ликвидность | $100 |
| Уровней стакана | 5 |
| Рынков за цикл | 15 (ротация для rate limit) |

**Что делает**:
1. Получает список рынков из Gamma API (пагинация, до 10 страниц × 100)
2. Фильтрует: ликвидность > $100, цена 2-98%, спред 0.5-20¢
3. Каждый цикл обрабатывает 15 рынков (ротация)
4. Для каждого: получает стакан из CLOB API, сохраняет snapshot в `data/live/<slug>/stream.jsonl`
5. Сохраняет мета-информацию в `data/live/<slug>/meta.json`

**Формат stream.jsonl**:
```json
{"ts":1718100000000,"type":"snap","tokenId":"abc...","bids":[{"p":0.65,"s":500}],"asks":[{"p":0.67,"s":300}],"mid":0.66,"spread":0.02}
```

**HTTP API**:
- `GET /status` — статус коллектора
- `GET /markets` — список рынков
- `POST /start` — начать сбор
- `POST /stop` — остановить сбор

### 2. Paper Trading Engine (Port 3002)

**Файл**: `mini-services/paper-trading/index.ts`

Основной движок торговли — реализует стратегию v5.0 Informed Edge Trading.

| Параметр | Значение |
|----------|----------|
| Порт | 3002 |
| Интервал цикла | 60 секунд |
| Максимум рынков | 30 |
| Мин. edge | 5¢ |
| Выход по edge | 2¢ |
| Kelly fraction | 25% (¼ Kelly) |
| Макс. позиция | $30/рынок |
| Мин. позиция | $5/рынок |
| Начальный баланс | $1,000 |
| Резерв банкролла | 20% |
| Макс. просадка (CB) | 25% |

**Жизненный цикл позиции**:
1. **ENTRY**: Edge > 5¢ → Kelly sizing → BUY_YES или BUY_NO → VWAP execution
2. **HOLD**: Edge валидный → ждём
3. **EXIT** (одно из):
   - Edge reversal (оценка развернулась на 3¢+)
   - Edge narrow (< 2¢)
   - Time exit (> 48 циклов)
   - Settlement avoid (< 24ч до погашения)

**State persistence**: `db/paper-trading-state.json`

**HTTP API**:
- `GET /` — текущий статус + все данные
- `POST /` с `{action: "start"}` — запустить
- `POST /` с `{action: "stop"}` — остановить
- `POST /` с `{action: "reset"}` — сбросить состояние

## Известные проблемы

1. **Дашборд не работает без AI**: Мини-сервисы запускаются вручную через `run-all.sh`, но не как systemd-сервисы. Когда AI-ассистент неактивен, процессы могут быть остановлены.
2. **Cross-market signals**: Заглушка — не реализована корреляция между рынками.
3. **LLM reliability**: AI-оценка зависит от доступности z-ai-web-dev-sdk.
4. **Win rate 33%**: Без LLM подтверждения винрейт низкий. С LLM ожидается > 50%.
5. **Монолитный page.tsx**: ~1200 строк в одном файле, стоит разбить на компоненты.
